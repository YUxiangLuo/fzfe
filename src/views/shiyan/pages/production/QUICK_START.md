### 1. 渐进式学习体验

```
左侧：概念学习             右侧：MPS表格（始终可见）

│ [计算公式]   │           │ ... │  ...  │... │
│ [下一步]     │           └────────────────────┘
└──────────────┘           第2期第1列已填充！
```

### 2. 智能状态管理

- **渐进式状态**：追踪当前步骤和已完成步骤
- **第2期数据**：每个字段独立管理，自动填充
- **完整计划**：步骤9生成所有期的MPS表

### 3. 统一计算逻辑

**服务水平公式（已修正）**：
```typescript
// ✅ 正确：基于预测需求
serviceLevel = 1 - (stockout / demandForecast);

// ❌ 错误：基于实际需求（旧版本的问题）
// serviceLevel = 1 - (stockout / actualDemand);
```

**为什么修正**：
- MPS是**事前计划**工具，所有计算都应基于预测数据
- 缺货量是基于预测需求计算的，服务水平也应基于预测需求
- 保持逻辑一致性，不引入"实际需求"概念

## 🐛 可能的问题与解决

### 问题1：步骤组件导入失败

**症状**：浏览器控制台报错 "Cannot find module"

**解决**：
- 检查所有步骤组件文件是否都已创建（ConceptStep1-9.tsx）
- 检查文件名大小写是否正确
- 重启开发服务器

### 问题2：API调用失败

**症状**：步骤9生成时报错

**解决**：
- 检查 `apiClient` 是否正确导入
- 检查 API 地址是否正确：`/model/predict`
- 检查模型类型映射 `MODEL_TYPE_MAP` 是否包含所需模型
- 测试时可以先注释掉API调用，使用模拟数据

### 问题3：表格数据不更新

**症状**：学习步骤完成后，表格数据没有变化

**解决**：
- 检查 `fillPeriod2Field()` 是否正确调用
- 检查 `useEffect` 依赖数组是否正确
- 使用浏览器React DevTools查看Context状态

### 问题4：步骤无法切换

**症状**：点击步骤导航没有反应

**解决**：
- 检查 `goToStep()` 方法是否正确实现
- 检查步骤状态（completed/current/locked）逻辑
- 确认已完成的步骤才能被点击
